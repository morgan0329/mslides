框架轻重的选择

轻的框架（被称为libary）

zepto.js: http://zeptojs.com/ 或 http://www.zeptojs.cn/

从源代码上看，zepto是基于jquery lite，专门正对移动端写的一套很轻量的js library.

hammer.js: 主要是封装移动端的Js或者是touch事件的一个library. http://hammerjs.github.io/

MV* Framework

Angular.js: Google 2009年出来的 http://angularjs.org/ or http://docs.angularjs.cn/api

backbone.js(和underscore联用的）: 2010年，出来的一套专注于做客户端mvc的框架。经过5年不断升级已经比较成熟了。http://backbonejs.org/

Ember.js: 用handlerbar作为client template. http://emberjs.com/

React.js: Facebook 这个框架听说比Angular好很多，但是比较新还不太成熟。http://facebook.github.io/react/

knockout.js: http://knockoutjs.com/

这里有很多MV* Framework比较：http://todomvc.com/

重的框架（fullstack） Mobile Optimized

Jquery Mobile: 这是一个很重的框架，修改了Jquery的代码，同时加入了自己的整套UI框架。只不过他的UI样式显得有些旧。

famo.us+angularjs: http://famo.us/?new

onsen UI: 用bower作为包管理的工具。跟ionic很相近，也是提供全套解决方案，同时页面扁平化，现代感。http://onsen.io/

framework7: 用handlerbar作为client template. 用requirejs作为包加载工具。页面做得很现代化。 http://www.idangero.us/framework7/

ali mui: http://spmjs.io/docs/alice-mobile/

ionic+angularjs+cordova: http://www.ionicframework.com/

Mobile Angular UI: 它是集成bootstrap + angular Js的移动端框架。 http://mobileangularui.com/

补充一点知识，拿Java做对比。

Java中的包依赖管理: maven VS npm或者bower

Java中的选择性地加载: 类开头的import语句 VS requirejs或者seajs

Java中的打包工具：maven,ant,ivy VS grunt或者gulp

Java中的servlet容器：tomcat/jboss VS nodejs的http server

Java中的页面模板:Velocity,Freemarker,Thymeleaf VS handlerbar,而类似于angular就不需要什么模板了

一个成熟的Web开发语言都差不多，都会有这几个要素的。
